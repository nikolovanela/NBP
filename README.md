# Redis Yelp Key-Value Project

Redis part of the university NoSQL Databases project.  
Uses the Yelp dataset subset and the same 10 query scenarios as the Oracle NoSQL team, enabling direct performance comparison.

## Technologies

- Docker Desktop
- Redis (port 6379)
- Python 3
- `redis`, `pandas`, `tqdm`, `matplotlib`
- Yelp Academic Dataset

## Step-by-step

### 1. Test the connection

```bash
python scripts/test_redis.py
```

Expected output:
```
Redis connection OK.
Test key written and read back:
{'message': 'Redis works!'}
Test key deleted. All good.
```

### 2. Download the Yelp dataset

Download from Kaggle: https://www.kaggle.com/datasets/yelp-dataset/yelp-dataset

Place these 3 files in `data/raw/`:
```
data/raw/yelp_academic_dataset_business.json
data/raw/yelp_academic_dataset_review.json
data/raw/yelp_academic_dataset_user.json
```

### 3. Prepare the subset

```bash
python scripts/prepare_yelp_subset.py
```

Filters: **Philadelphia restaurants only**. Max 3,000 businesses, 50,000 reviews.

Output:
```
data/processed/business_subset.jsonl
data/processed/review_subset.jsonl
data/processed/user_subset.jsonl
```

### 4. Import into Redis

```bash
python scripts/import_yelp_redis.py
```

Imports all data using a **two-level key-value model**:

**Level 1 — basic entities:**
```
business:{business_id}
user:{user_id}
review:{review_id}
```

**Level 2 — aggregated / denormalized keys:**
```
business_reviews:{business_id}    ← list of review IDs per business
user_reviews:{user_id}            ← list of review IDs per user
stats:business:{business_id}      ← pre-computed review count + avg stars
city_category:Philadelphia:Restaurants  ← index of all restaurant IDs in city
```

### 5. Run all 10 query scenarios

```bash
python scripts/query_redis.py
```

To save output to a file:
```bash
python scripts/query_redis.py | tee results/redis_query_examples.txt
```

#### Query categories

| # | Query | Category |
|---|-------|----------|
| Q1 | Restaurants in city | Simple |
| Q2 | Business by ID | Simple |
| Q3 | Review IDs for business | Simple |
| Q4 | Business stats | Simple |
| Q5 | Top reviews for business (pipeline join) | Complex |
| Q6 | User profile + their reviews (join) | Complex |
| Q7 | Top businesses by avg stars (full scan + sort) | Aggregated |
| Q8 | Star-rating distribution for a business | Aggregated |
| Q9 | Most active reviewers in city | Aggregated |
| Q10 | Rating-band distribution for all businesses | Aggregated |

### 6. Measure performance

```bash
python scripts/performance_redis.py
```

Runs each of the 10 queries **100 times** and saves avg/min/max times.  
Output: `results/redis_performance.csv`

### 7. Generate Redis performance chart

```bash
python scripts/plot_redis_performance.py
```

Output: `results/redis_performance_chart.png`  
Bars are **color-coded by category**: blue = simple, orange = complex, green = aggregated.

### 8. Prepare comparison CSV

```bash
python scripts/prepare_redis_comparison.py
```

Output: `results/redis_performance_for_comparison.csv`

### 9. Generate Oracle NoSQL vs Redis comparison chart

Place `oracle_performance_for_comparison.csv` (from Oracle team) in `results/`, then:

```bash
python scripts/plot_comparison.py
```

Output: `results/comparison_chart.png`

## Key-Value Data Model

Redis stores everything as JSON strings:

```python
r.set("business:MTSW4McQd7CbVtyjqoe9mw", json.dumps({ ... }))
value = json.loads(r.get("business:MTSW4McQd7CbVtyjqoe9mw"))
```

This mirrors the Oracle NoSQL model exactly, making performance comparison valid.

---

## Notes

- Redis runs on port **6379**, not the default 6379.
- Raw Yelp data is not committed to git (files are too large).
- `data/raw/` and `data/processed/` are excluded via `.gitignore`.
- Aggregated queries (Q7, Q9, Q10) perform in-memory computation in Python since Redis has no native aggregation engine — this is an intentional design trade-off documented in the elaborat.
